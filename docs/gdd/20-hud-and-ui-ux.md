# 20. HUD and UI/UX

## Race HUD

The HUD should include:

- speed
- gear
- position
- lap
- lap timer
- best lap
- nitro meter
- damage
- cash delta
- simplified minimap or progress strip
- weather icon
- grip hint

## Pre-race screen

Must show:

- track name
- tour name
- weather
- laps
- track difficulty
- recommended tires
- current standings
- cash and repair estimate
- selected car and setup summary

## Results screen

Must show:

- finishing order
- points earned
- cash earned
- bonuses
- damage taken
- fastest lap
- next race card

## Garage

Tabs:

- Car
- Upgrades
- Repairs
- Tires
- Setup
- Tour standings
- Stats / ghost

## Upgrade shop

Priorities:

- one-button compare
- clear “current vs next”
- explanation of what each upgrade changes
- compatibility and install lockouts shown inline

## Settings

- controls
- display
- audio
- accessibility
- game feel assists
- profile / save clear

## Pause menu

- resume
- restart race
- retire race
- settings
- leaderboard / ghost
- exit to title

## Save and load

- Auto-save after each race and each garage purchase
- Manual profile export / import
- Versioned save migrations

## UX wireframe descriptions

Race HUD layout

- Top-left: lap, position, tour points
- Top-center: current lap timer
- Top-right: best lap / ghost delta
- Bottom-left: damage and weather grip
- Bottom-center: nitro meter
- Bottom-right: speed and gear

Garage layout

- Left column: car and damage
- Center: upgrade list
- Right column: current cash, forecast, standings, next race info

Results layout

- Top banner: finish result
- Center split: standings left, cash and bonuses right
- Bottom CTA row: repair, upgrade, next race
