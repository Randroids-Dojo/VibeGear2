# 19. Controls and input

## Keyboard layout

Default keyboard bindings should preserve VibeRacer’s current clarity—accelerate, brake/reverse, left, right, handbrake—with remapping as a first-class feature, because the current repo already persists control settings and keyboard input via local storage and hooks. [21]

Default layout

| Action | Default |
| --- | --- |
| Accelerate | Up / W |
| Brake / reverse | Down / S |
| Steer left | Left / A |
| Steer right | Right / D |
| Nitro | Space / X on pad |
| Handbrake | Shift / A on pad |
| Pause | Esc / Start |
| Shift up manual | E / RB |
| Shift down manual | Q / LB |

## Gamepad layout

| Action | Pad |
| --- | --- |
| Accelerate | Right trigger |
| Brake / reverse | Left trigger |
| Steer | Left stick |
| Nitro | X / Square |
| Handbrake | A / Cross |
| Shift up | RB |
| Shift down | LB |
| Pause | Start |

## Remapping

Full remapping is required on desktop. Store control profiles locally and optionally in save data.

## Accessibility controls

- Auto-accelerate option
- Brake assist
- Steering smoothing
- Hold-to-nitro or toggle nitro
- Reduced simultaneous input mode
- Visual-only weather mode

## Touch and mobile future work

VibeRacer already has single-stick and dual-stick touch control concepts. VibeGear2 should keep the idea as future work but not let it shape core handling decisions before desktop feel is locked. [22]
